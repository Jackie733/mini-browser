# Chapter 2: Drawing to the Screen - Exercises Note

This document logs the implementation details, key concepts, and fixes for the exercises of Chapter 2 (Drawing to the Screen).

## Added
- **Exercise 2-1: Line breaks**
  - `layout()` flushes the current line and advances `cursor_y` when encountering a newline character.
  - Paragraph breaks produce extra vertical space beyond `VSTEP`.

- **Exercise 2-2: Mouse wheel**
  - Keyboard bindings: `<Up>` / `<Down>` scroll by `SCROLL_STEP`.
  - Mouse wheel: `<MouseWheel>` (macOS/Windows) and `<Button-4>` / `<Button-5>` (Linux) events bound to the same handler.
  - Scroll clamped to document bounds.

- **Exercise 2-3: Resizing**
  - `canvas.pack(fill="both", expand=True)` allows the canvas to fill the window.
  - `<Configure>` event triggers `resize()`: re-lays out text with new `width`/`height` and redraws.
  - Only re-layouts when dimensions actually changed.

- **Exercise 2-4: Scrollbar**
  - Blue rectangle drawn at the right edge of the screen.
  - Position and height proportional to scroll offset and viewport-to-document ratio.
  - Hidden when the whole document fits on screen.

- **Exercise 2-5: Emoji**
  - Loads OpenMoji PNG files from `emojis/` directory by Unicode code point.
  - Images cached in `self.emoji_cache` to avoid repeated disk reads.
  - Drawn via `create_image` instead of `create_text`.

- **Exercise 2-6: `about:blank`**
  - Special-cased in `Browser.load()` to render an empty page.
  - Malformed URLs caught by `try/except`, falling back to blank page.

## Key Ideas
- **Display list**: an intermediate representation `[(type, x, y, value), ...]` decoupling layout from drawing.
- **Scrolling**: offset `self.scroll` subtracts from `y` during drawing; items outside the viewport culled by `y` range check.
- **Coordinate system**: `WIDTH x HEIGHT` canvas; `HSTEP`/`VSTEP` provide margins and line spacing.
- **RTL support**: `is_rtl_char()` detects bidirectional characters; RTL lines are right-aligned with characters drawn right-to-left via `place_line`.

## Bugs / Fixes
- **Scroll overflow**: display list items below the visible area still added; clamped via `clamp_scroll()` using the last item's `y`.
- **Image sizing**: emoji loaded at native PNG size (16×16), not scaled to match text metrics; may appear misaligned on different font sizes.
