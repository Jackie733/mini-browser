# Chapter 9: Running Interactive Scripts

## Added

- `dukpy` executes JavaScript snippets and page scripts.
- `JSContext` owns one `dukpy.JSInterpreter` per tab.
- `src/runtime.js` defines small browser-like JavaScript objects.
- `console.log` calls back into Python's `print`.
- External `<script src="...">` files are loaded and run during `Tab.load`.
- `document.querySelectorAll` calls into Python selector matching.
- Python DOM nodes are represented in JavaScript by integer handles.
- JavaScript wraps handles in lightweight `Node` objects.
- `Node.getAttribute` passes the handle back to Python to read attributes.

## Key Ideas

- DukPy is only a JavaScript engine; browser APIs like `document` must be added.
- `export_function` exposes selected Python functions to JavaScript.
- Handles avoid trying to pass Python `Element` objects directly into JavaScript.
- One `JSContext` per tab lets multiple scripts on the same page share variables.

## Bugs / Fixes

- Local files should be tested through a local HTTP server, not `./index.html`.
- Page scripts should run through `JSContext`, not bare `dukpy.evaljs`, so the
  runtime objects are available.

## Python Notes

- Dictionaries are used in both directions: node to handle, and handle to node.
- `os.path.dirname(__file__)` makes `runtime.js` load relative to `browser.py`.
