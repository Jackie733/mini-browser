# Chapter 1: URLs and HTTP - Exercises Note

This document logs the implementation details, key concepts, and fixes for the exercises of Chapter 1 (URLs and HTTP).

## Added
- **Exercise 1-1: HTTP/1.1**
  - Added `Connection: keep-alive` and `User-Agent: AntigravityBrowser/1.0` headers to requests.
  - Made headers extensible via a `dict` passed to `request(headers=...)`.

- **Exercise 1-2: File URLs**
  - Added support for the `file` scheme (`file:///path`).
  - Defaults to opening `default.html` when no URL is provided.

- **Exercise 1-3: `data` scheme**
  - Added support for `data:text/html,...` inline content.
  - Uses `urllib.parse.unquote` to decode the embedded text.

- **Exercise 1-4: Entities**
  - In `show()`, replaces `&lt;` with `<` and `&gt;` with `>` before printing.

- **Exercise 1-5: `view-source` scheme**
  - Added `view-source:` protocol that fetches the inner URL and escapes `<`/`>` so raw HTML is displayed.

- **Exercise 1-6: Keep-alive**
  - Implemented `SOCKET_CACHE` dictionary keyed by `(host, port)` to reuse TCP connections.
  - On connection error (stale socket), falls back to creating a new socket.
  - Closes and removes cached socket when server sends `Connection: close`.

- **Exercise 1-7: Redirects**
  - Handles 3xx status codes by following the `Location` header.
  - Resolves relative `Location` values (starting with `/` or no scheme) against the original URL.
  - Enforces a `redirect_limit` (default 10) to prevent infinite redirect loops.

## Key Ideas
- **URL structure**: scheme, host, port, path parsed manually via string splitting.
- **Socket reuse**: TCP connections cached to avoid repeated SSL handshakes.
- **Chunked transfer encoding**: body read in chunks of `[size]\r\n[data]\r\n`.
- **Gzip decompression**: transparently handled via `gzip.decompress`.
- **HTTP caching**: respects `Cache-Control` with `max-age` and `no-store`.

## Bugs / Fixes
- **Stale sockets**: Cached socket may be closed by server; catch `OSError` and reconnect.
- **Relative redirects**: `Location` may omit scheme/host (`/path`) or be a full URL; both cases handled.
- **Content-Length vs chunked**: response body read strategy chosen based on `transfer-encoding` or `content-length` headers; fallback reads all remaining data.
