# Chapter 3: Formatting Text - Exercises Note

This document logs the implementation details, key concepts, and fixes for the exercises of Chapter 3 (Formatting Text).

## Added
- **Exercise 3-1: Centered text**
  - Added support for the `<h1 class="title">` tag to center-align heading lines.
  - Implemented by calculating the remaining width of the line during `flush()` and offsetting the x-coordinates of all words in the line by `remaining_width / 2`.
  - Added a state flag `self.is_centered` to toggle alignment.

- **Exercise 3-2: Superscripts**
  - Added support for the `<sup>` tag.
  - Implemented smaller text size (`self.size -= 6`) and aligned the top of the superscript letters to the top of normal letters (`y = baseline - max_ascent`).
  - Added state flag `self.is_superscript` and stored it in the `self.line` layout queue to track layout behavior at the word level.

- **Exercise 3-3: Soft hyphens**
  - Added support for the soft hyphen character `\u00ad` (`\xad`).
  - Implemented greedy line-breaking inside `Layout.word()`. When a word overflows, it tries to break at the rightmost soft hyphen where the prefix plus a visible hyphen `-` fits on the current line.
  - The prefix is drawn with a trailing hyphen and the line is flushed, while the rest of the word is processed recursively on a new line.

- **Exercise 3-4: Small caps**
  - Added support for the `<abbr>` tag.
  - Split words inside `<abbr>` into runs of lowercase and non-lowercase characters using `itertools.groupby`.
  - Lowercase characters are capitalized, small-sized (`size - 2`), and bolded, while other characters use the normal font. The runs are drawn consecutively with no intermediate spaces.

- **Exercise 3-5: Preformatted text**
  - Added support for the `<pre>` tag.
  - Preserved whitespace (spaces and newlines) by processing characters individually instead of using `split()`.
  - Prevented automatic line-wrapping (`self.in_pre`) and used Courier as the fixed-width font.
  - Ensured nested tags (like `<b>`) and empty line breaks work correctly inside preformatted text.

## Key Ideas
- **Inline vs Block-level Elements**: Tags like `abbr` and `sup` are inline and do not trigger line flushes, while `pre` is block-level and requires flushes on entry/exit.
- **Baseline Alignment**: Text is aligned to a baseline. For superscripts, shifting the drawing coordinate `y` to `baseline - max_ascent` effectively aligns the tops.
- **Run-based Rendering**: Complex inline rendering (like small caps) requires splitting a single word into runs of different fonts/styles and laying them out sequentially.

## Bugs / Fixes
- **Stale Tkinter Windows**: Stale background python browser processes could cause visual confusion in VNC. Resolved by cleaning up processes between runs.
- **Double quote mismatch**: Ensured HTML tag parser correctly handles double quotes `<h1 class="title">` instead of just single quotes.
- **Empty line flush**: Extended `Layout.flush()` to advance `self.cursor_y` even if `self.line` is empty when `self.in_pre` is active, supporting consecutive newlines.
